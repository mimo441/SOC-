# TCP (Transmission Control Protocol)

## Что это

**TCP** - это транспортный протокол(находится на 4 уровне OSI), обеспечивающий надёжную, упорядоченную и проверяемую передачу данных между узлами сети, передаёт данные в виде сегментов (segments).

TCP используется в:

- HTTP/HTTPS
- SSH
- FTP
- SMTP
- и других прикладных протоколах

Основная цель TCP - гарантированная доставка данных с контролем ошибок.


## Как работает TCP

**3-Way Handshake (трёхстороннее рукопожатие)** - это процесс установки TCP соединения.

Перед началом передачи данных выполняется процесс установления соединения:

1. **SYN** - клиент отправляет запрос на подключение "Хочу соединение"
2. **SYN/ACK** - сервер подтверждает получение запроса "Хорошо соединяемся"
3. **ACK** - клиент подтверждает соединение "Соединение установлено"

После этого можно отправлять данные (HTTP/HTTPS).


### Передача данных

Данные разбиваются на части, каждая часть содержит:

  - Sequence Number
  - ACK (подтверждение)
  - Checksum

**Retransmission** - повторная отправка потерянного пакета,если он будет потерян.

TCP также использует:

- Flow Control
- Congestion Control


### Завершение соединения


Обычно процесс выглядит так:

- Клиент -> FIN
- Сервер -> ACK
- Сервер -> FIN
- Клиент -> ACK

В некоторых случаях соединение может быть принудительно закрыто через **RST**.


### Основные функции TCP

- Надёжная доставка данных
- Контроль целостности (checksum)
- Повторная передача потерянных сегментов
- Контроль перегрузки сети
- Управление потоком данных

## Атаки

### SYN Flood

Что это: атака, при которой отправляется большое количество SYN-пакетов для перегрузки сервера.

Принцип работы: злоумышленник отправляет SYN -> сервер отвечает SYN/ACK -> но клиент не отправляет ACK.  
В итоге сервер держит много "полуоткрытых" соединений и не может обслуживать нормальных пользователей.

Признаки в логах:

- резкий рост SYN
- мало ACK
- много half-open connections
- переполнение очереди соединений

Что делает SOC аналитик:

- фиксирует аномалию трафика
- проверяет источник (один IP или DDoS)
- включает фильтрацию (rate limit)
- может блокировать IP


### Port Scanning

Что это: разведка сети для поиска открытых портов и доступных сервисов.

Принцип работы: злоумышленник отправляет запросы (SYN) на разные порты и анализирует ответ:

- SYN/ACK -> порт открыт
- RST -> порт закрыт

Признаки в логах:

- один IP обращается к множеству портов
- много RST ответов
- короткие соединения без передачи данных

Что делает SOC аналитик:

- определяет источник сканирования
- проверяет частоту запросов
- может заблокировать IP
- классифицирует как reconnaissance


### TCP Reset

Что это: атака или вмешательство, при котором отправляется RST-пакет для принудительного разрыва соединения.

Принцип работы: злоумышленник отправляет поддельный RST, заставляя одну из сторон закрыть соединение.

Признаки в логах:

- неожиданные RST
- разрыв активных соединений
- пользователи жалуются на обрывы

Что делает SOC аналитик:

- анализирует источник RST
- проверяет, не является ли это MITM-атакой
- проверяет сеть на аномалии
- контроль доставки пакетов в нужном порядке
- ищет повторяющиеся паттерны

Примечание: не каждый RST - это атака. Часто это нормальное поведение TCP.
