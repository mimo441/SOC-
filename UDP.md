# UDP (User Datagram Protocol)

## Что это
UDP - транспортный протокол (находится на 4 уровне OSI), который обеспечивает быструю передачу данных без установления соединения.

В сравнении с TCP у UDP:
- нет гарантии доставки
- нет порядка пакетов
- нет повторной передачи

Основная цель - минимальная задержка и высокая скорость.



## Где используется
- DNS
- VoIP (интернет-звонки)
- стриминг
- онлайн-игры



## Как работает UDP

UDP - connectionless протокол (без соединения).

Нет handshake:
- данные отправляются сразу

Процесс:
- 1. Формируется UDP datagram (датагоамма)
- 2. Указывается IP и порт
- 3. Пакет отправляется в сеть



## Структура UDP

UDP пакет (датаграмма) содержит:
- Source Port
- Destination Port
- Length
- Checksum



## Особенности UDP

- высокая скорость
- низкая задержка
- нет контроля доставки
- нет управления перегрузкой
- нет повторной отправки



## Атаки

### UDP Flood

Что это: атака, при которой отправляется большое количество UDP-пакетов для перегрузки сервера или сети.

Принцип работы:
-Злоумышленник отправляет UDP-пакеты на случайные порты.
-Сервер пытается обработать их или отвечает ICMP (port unreachable).

Признаки в логах:
- большой объём UDP трафика
- пакеты на случайные порты
- высокая нагрузка на сеть

Что делает SOC:
- анализирует источник трафика
- проверяет распределённость (DDoS)
- применяет rate limiting
- блокирует IP



### DNS Amplification

Что это: отражённая DDoS-атака с усилением через DNS.

Как работает:
-Злоумышленник подменяет IP жертвы и отправляет запросы к DNS-серверам.
-Сервер отправляет большой ответ жертве.

Признаки в логах:
- большой исходящий DNS-трафик
- несоответствие запросов и ответов
- UDP порт 53

Что делает SOC:
- анализирует DNS-трафик
- проверяет spoofing
- ограничивает ответы
- использует фильтрацию



### UDP Scanning

Что это: сканирование UDP портов для поиска сервисов.

Как работает:
- Отправляются UDP-пакеты:
- нет ответа, значит порт открыт или фильтруется
- ICMP unreachable -> порт закрыт

Признаки в логах:
- UDP на разные порты
- ICMP ответы
- отсутствие ответа от некоторых портов

Что делает SOC:
- анализирует поведение источника
- ищет аномалии
- классифицирует как reconnaissance



## Признаки в логах

### Аномальный UDP трафик
- резкий рост UDP пакетов
- неизвестные IP
- нестандартные порты



### Отсутствие ответа
- может означать открытый порт
- или фильтрацию



### DNS аномалии
- длинные домены
- частые запросы
- подозрение на DNS tunneling



## Примечание

UDP сам по себе не шифрует данные.
Без дополнительных протоколов трафик может быть перехвачен.

Для защиты используются:
- VPN
- DNS over HTTPS (DoH)
- DNS over TLS (DoT)
